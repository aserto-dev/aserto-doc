A comparison predicate for filtering on string fields.

At most one field may be specified.

```graphql
input StringWhere {
  contains: String
  startsWith: String
  endsWith: String
  equals: String
  notEquals: String
  in: [String!]
  notIn: [String!]
}
```


### Fields

#### [`contains`](#) ([`String`](/docs/directory-api-reference/schema/scalars/string))

The field being compared contains the given value as a substring.


#### [`startsWith`](#) ([`String`](/docs/directory-api-reference/schema/scalars/string))

The field being compared contains the given value as a prefix.


#### [`endsWith`](#) ([`String`](/docs/directory-api-reference/schema/scalars/string))

The field being compared contains the given value as a suffix.


#### [`equals`](#) ([`String`](/docs/directory-api-reference/schema/scalars/string))

The field being compared is equal to the given value.


#### [`notEquals`](#) ([`String`](/docs/directory-api-reference/schema/scalars/string))

The field being compared is not equal to the given value.


#### [`in`](#) ([`[String!]`](/docs/directory-api-reference/schema/scalars/string))

The field being compared is equal to one of the given values..


#### [`notIn`](#) ([`[String!]`](/docs/directory-api-reference/schema/scalars/string))

The field being compared is not equal to any of the given values.
